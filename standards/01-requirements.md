## 2. 需求清单

### US-1 初始化项目工程化与 CI/CD · 状态: Done

作为 **项目开发者**,我想要 项目具备基础工程结构、测试、CI 与 CD,以便 每次开发都能自动检查并自动部署。

验收标准:
- AC1: 从 `main` 开 feature 分支完成初始化,不直接 push main。✅
- AC2: PR 触发 CI,包含格式、静态检查、单元测试+覆盖率、模型门禁、构建检查。✅
- AC3: CI 全绿后合并 main。
- AC4: 合并 main 自动触发 CD,部署后健康检查通过。
- AC5: 完成后更新 `PROGRESS.md`。✅

### US-2 模块1 · 交互式数据探索仪表盘 · 状态: Done

作为 **数据分析师**,我想要 一个可动态筛选与可视化的仪表盘,以便 快速看清各字段与认购率的关系。

验收标准:
- AC1: 对所有字段动态筛选(数值滑块 + 类别多选)。✅
- AC2: Plotly 可视化:数值分布、按 subscribe 分组对比、相关性热图。✅
- AC3: 输出特征洞察文字报告。✅
- AC4: 计算逻辑为纯函数并有单测(`insights.py`)。✅

### US-3 模块2 · 机器学习模型训练 · 状态: Done

作为 **建模工程师**,我想要 训练多模型并选出 AUC 最佳者保存,以便 在线服务直接加载。

验收标准:
- AC1: 预处理(类别 OneHot + 数值标准化),剔除 `duration` 泄漏与 `id`。✅
- AC2: 训练 LR / RF / GBDT,按验证集 AUC 选最佳。✅
- AC3: 类别不平衡用 AUC/P/R/F1 评估,而非准确率。✅
- AC4: 最佳模型保存到 `models/model.pkl`。✅
- AC5: `python -m src.model --check-auc 0.80` 不达标退出码=1(CI 门禁)。✅

### US-4 模块3 · 在线预测服务 · 状态: Done

作为 **营销系统**,我想要 一个 REST API 输入客户特征返回认购预测,以便 实时给客户打分。

验收标准:
- AC1: `POST /predict` 接收 19 个特征,返回 subscribe + 概率 + 置信度。✅
- AC2: `POST /predict/batch` 支持批量。✅
- AC3: `GET /health` 健康检查(CD 用)。✅
- AC4: 加载 `models/model.pkl`;无模型时返回 503 而非崩溃。✅
- AC5: 有单测(TestClient 覆盖正常/缺字段/空批量)。✅

## 3. 非功能需求

- **安全**:密钥只进 Secrets;数据与模型不进 Git。
- **可复现**:固定随机种子;锁依赖版本;统一 conda 环境 `envbank`(Python 3.11)。
- **可维护**:一需求一小 PR;业务逻辑与 I/O 分离。
- **可部署**:部署后必须 `/health` 健康检查通过。
